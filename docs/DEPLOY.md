# Deployment

Three pieces: **backend** (Render), **web** (Vercel), **mobile** (APK / sideload).
All free-tier friendly.

## Backend → Render (free)
1. Push the repo to GitHub (done).
2. Render → **New → Blueprint** → pick this repo. It reads `backend/render.yaml`
   and provisions a free web service (build `pip install -r requirements.txt`,
   start `uvicorn app.main:app --host 0.0.0.0 --port $PORT`).
3. After first deploy, open the service → **Environment** and set:
   - `SECRET_KEY` (auto-generated by the blueprint; or set your own)
   - optional `LLM_PROVIDER`/`LLM_MODEL`/`LLM_API_KEY`, `VOICE_*` (see ENV_SETUP.md)
4. Health check: `https://<your-service>.onrender.com/api/v1/health`.
5. **Persistence:** the free instance has an ephemeral disk (SQLite resets on
   redeploy, and re-seeds the sample data on boot). For durable data, add a free
   **Render Postgres**, set `DATABASE_URL` to its Internal URL, and add
   `psycopg2-binary` (it's in `requirements-postgres.txt`).

> Free instances sleep when idle and take ~30s to wake on the first request.

## Web → Vercel (free)
1. Vercel → **Add New → Project** → import this repo.
2. Set **Root Directory = `web`** (important).
3. Add env var `NEXT_PUBLIC_API_BASE` = your Render URL
   (e.g. `https://setu-backend.onrender.com`).
4. Deploy. You get an installable PWA at `https://<project>.vercel.app`.
5. Back on Render, set `CORS_ORIGINS` to your Vercel URL (or keep `*` for the demo).

## Mobile → Android APK
On the mobile dev's machine (Flutter installed):
```bash
cd mobile
flutter create . --org app.setu --platforms=android,ios   # first time only
flutter pub get
# add permissions per mobile/README.md, then:
flutter build apk --release --dart-define=API_BASE=https://setu-backend.onrender.com
```
The APK is at `build/app/outputs/flutter-apk/app-release.apk` - share it to
install on any Android phone. iOS uses the same code via `flutter build ipa`.

## Local cross-team dev → Docker Compose
Spin up backend + web together with one command:
```bash
docker compose up --build
# backend → http://localhost:8000   web → http://localhost:3000
```
See `docker-compose.yml` at the repo root. Put any keys in `backend/.env` first.
